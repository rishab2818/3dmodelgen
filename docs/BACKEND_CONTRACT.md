# Backend Contract

Status: **Draft v1 — for sign-off before M1**

Defines the wire formats between (a) Tauri ↔ Python orchestrator and (b) Python orchestrator ↔ GPU backend. Implement this; do not freelance.

All schemas are Pydantic v2 on the Python side, mirrored as zod schemas on the TS side (auto-generated by `scripts/gen_ts_schemas.py`).

---

## 1. Tauri ↔ Python orchestrator

**Transport:** HTTP + Server-Sent Events. Backend binds to `127.0.0.1:7878` (configurable via `BACKEND_PORT`).

### 1.1 REST endpoints

| Method | Path | Purpose |
|---|---|---|
| `GET`  | `/health` | liveness probe. returns `{status, gpu_backend, gpu_status}` |
| `GET`  | `/config` | read effective config |
| `PUT`  | `/config` | mutate config (e.g. switch GPU backend mode) |
| `POST` | `/jobs` | create a new job |
| `GET`  | `/jobs` | list jobs (paginated, default 50) |
| `GET`  | `/jobs/{id}` | get job snapshot |
| `POST` | `/jobs/{id}/pause` | request pause (distinct from cancel) |
| `POST` | `/jobs/{id}/resume` | resume a paused job |
| `POST` | `/jobs/{id}/cancel` | cancel a running job (terminal) |
| `DELETE`| `/jobs/{id}` | delete job + artifacts |
| `GET`  | `/jobs/{id}/events` | SSE stream of per-job events |
| `GET`  | `/jobs/{id}/artifacts/{path}` | fetch an artifact (sandboxed under `temp/{id}/`) |
| `GET`  | `/jobs/{id}/budget` | per-job GPU-time summary, call count, cache hits |
| `GET`  | `/budget` | system-wide GPU-time totals (today / week / lifetime, per-provider) |
| `POST` | `/cache/clear` | clear the generation cache; returns bytes freed |
| `GET`  | `/exports/{id}/{file}` | fetch a final export |
| `GET`  | `/models` | available generators + their availability for current backend |

### 1.2 `POST /jobs` request schema

```python
class CreateJobRequest(BaseModel):
    input_images: list[Path]            # absolute paths from Tauri (already validated)
    target_quality: float = 0.85
    max_iterations: int = 6
    seed: int = 42
    generator_pref: Literal["auto", "triposr", "hunyuan3d-2", "instantmesh"] = "auto"
    export_formats: list[Literal["glb", "obj", "ply"]] = ["glb", "obj", "ply"]
    label: str | None = None             # human-readable name for UI
```

### 1.3 Job snapshot schema

```python
class JobSnapshot(BaseModel):
    id: UUID
    label: str
    state: Literal[
        "queued", "running",
        "paused_by_user", "paused_remote_offline",
        "paused_crashed", "paused_budget",
        "succeeded", "failed", "cancelled",
    ]
    paused_reason: str | None             # human-readable, set when state startswith "paused_"
    created_at: datetime
    updated_at: datetime
    current_iteration: int
    best_iteration: int | None
    iterations: list[IterationRecord]
    error: ErrorRecord | None
    inputs: JobInputs                     # echoes the create request
    budget: JobBudgetSummary              # per §1.6 below
    exports: ExportPaths | None           # populated when state == succeeded

class IterationRecord(BaseModel):
    n: int
    started_at: datetime
    finished_at: datetime | None
    stages: list[StageRecord]
    score: EvaluationReport | None
    refinement_action: str | None        # what was decided after this iteration

class StageRecord(BaseModel):
    name: str                            # e.g. "generate.initial"
    status: Literal["pending", "running", "complete", "failed"]
    started_at: datetime | None
    finished_at: datetime | None
    progress: float
    artifacts: list[Path]
    error: ErrorRecord | None
```

### 1.4 SSE event types

```jsonc
// Sent on connection: snapshot of current state so the client doesn't need a separate GET.
{ "event": "snapshot", "data": { "job": <JobSnapshot> } }

// Per-stage tick.
{ "event": "stage_update",
  "data": { "iteration": 2, "stage": "render.multiview",
            "status": "running", "progress": 0.5, "message": "...", "artifacts": [], "elapsed_ms": 12345 } }

// Iteration complete with evaluation.
{ "event": "iteration_complete",
  "data": { "iteration": 2, "score": <EvaluationReport>, "refinement_action": "texture_refine_only" } }

// Terminal.
{ "event": "job_complete", "data": { "state": "succeeded", "exports": <ExportPaths> } }
{ "event": "job_failed",   "data": { "error": <ErrorRecord> } }

// Pause / resume.
{ "event": "job_paused",  "data": { "reason": "by_user" | "remote_offline" | "crashed" | "budget" } }
{ "event": "job_resumed", "data": { "resumed_at_stage": "generate.initial" } }

// Budget update — emitted after every stage that triggered a GPU call (real or cached).
{ "event": "budget_update",
  "data": { "stage": "generate.initial", "cached": false, "runtime_ms": 73420,
            "provider": "colab", "job_total_runtime_ms": 142000 } }

// Remote backend connectivity.
{ "event": "remote_offline", "data": { "since": "..." } }
{ "event": "remote_online",  "data": { "at": "..." } }

// Heartbeat every 15s so reverse-proxies / browsers don't kill the stream.
{ "event": "heartbeat", "data": { "ts": "..." } }
```

Clients **must** treat unknown events as no-ops (forward compatibility).

### 1.5 Error shape

```python
class ErrorRecord(BaseModel):
    code: str                            # stable identifier: PREPROC_FAILED, GENERATOR_OOM, etc.
    message: str                         # human-readable
    stage: str | None
    iteration: int | None
    details: dict | None                 # tool-stderr, etc.
```

Stable error codes (frontend pattern-matches on these):

| Code | Meaning |
|---|---|
| `PREPROC_FAILED` | preprocessing crashed |
| `PREPROC_EMPTY_MASK` | background removal removed everything |
| `GENERATOR_OOM` | generator ran out of memory |
| `GENERATOR_TIMEOUT` | generation took longer than the per-stage budget |
| `BLENDER_FAILED` | Blender subprocess returned non-zero |
| `EVAL_FAILED` | evaluator could not run |
| `REMOTE_GPU_OFFLINE` | remote backend is unreachable |
| `REMOTE_GPU_AUTH` | ngrok / Modal auth failed |
| `CANCELLED_BY_USER` | user clicked cancel |
| `DISK_FULL` | OS reported ENOSPC |
| `INTERNAL` | unhandled — has stack trace in `details.traceback` |
| `BUDGET_CAP_HIT` | per-job or per-session GPU-time cap exceeded; job is `paused_budget`, not failed |

### 1.6 Budget summary shape

```python
class JobBudgetSummary(BaseModel):
    total_runtime_ms: int            # sum of all gpu_call.runtime_ms for this job
    call_count: int
    cached_call_count: int
    cache_hit_rate: float            # cached_call_count / call_count (or 0 if no calls)
    by_provider: dict[str, int]      # {"colab": 142000, "modal": 0, ...}
    cap_seconds: int | None          # per-job cap if set
    cap_remaining_seconds: int | None
```

Returned by `GET /jobs/{id}/budget` and embedded in `JobSnapshot.budget` (§1.3). The system-wide `GET /budget` returns the same shape but with `total_runtime_ms` summed across all jobs and a `window` field of `"today" | "week" | "lifetime"`.

## 2. Python orchestrator ↔ GPU backend

**Transport:** HTTP. The remote endpoint runs **the exact same `ai_models/` adapter code** as a local one — packaged into a FastAPI app via `ai_models/remote_server.py`. The orchestrator never knows whether it's talking to localhost, ngrok, or Modal.

### 2.1 Endpoints

| Method | Path | Purpose |
|---|---|---|
| `GET`  | `/health` | `{status, models_loaded: [...], vram_free_mb}` |
| `POST` | `/generate` | run a generator |
| `POST` | `/refine_texture` | run texture-only refinement |
| `POST` | `/evaluate` | run the evaluator stack (CLIP+DINO+LPIPS) |
| `POST` | `/preload` | warm up a specific model (avoids cold-start during a job) |

### 2.2 `POST /generate`

```python
class GenerateRequest(BaseModel):
    model: Literal["triposr", "hunyuan3d-2", "instantmesh"]
    images_b64: list[str]                # base64-encoded PNG bytes
    seed: int
    params: dict                         # model-specific, validated by adapter

class GenerateResponse(BaseModel):
    mesh_obj_b64: str                    # gzip+base64 of OBJ bytes
    albedo_png_b64: str | None
    meta: GenerationMeta

class GenerationMeta(BaseModel):
    model: str
    revision: str                        # HuggingFace commit hash
    seed: int
    runtime_ms: int
    vram_peak_mb: int
    tri_count: int
```

### 2.3 `POST /evaluate`

```python
class EvaluateRequest(BaseModel):
    original_png_b64: str
    mask_png_b64: str
    renders: dict[str, str]              # view_name → base64 PNG
    weights: dict[str, float]            # per-metric weights, optional

class EvaluateResponse(BaseModel):
    report: EvaluationReport             # see QUALITY_RUBRIC.md §3
    runtime_ms: int
```

### 2.4 Why HTTP and not gRPC / WebSocket / RPC

- HTTP works over ngrok unchanged. gRPC over ngrok requires extra plumbing.
- We are not latency-bound. Calls are 10+ seconds; 200ms of HTTP overhead is irrelevant.
- Debuggable with `curl`. No mystery framing.

### 2.5 Auth

A shared static bearer token (`GPU_BACKEND_TOKEN` env on both sides). Rotated per dev session. Not a real security model — it just prevents random ngrok-fingerprint scanners from triggering inference.

### 2.6 Idempotency

Every `POST` to `/generate`, `/evaluate`, `/refine_texture` MUST carry an `Idempotency-Key` header. Format:

```
Idempotency-Key: {job_id}:{iteration}:{stage}:{call_index}
```

The remote endpoint maintains an LRU (default 1000 entries, 24h TTL) mapping `key → (status_code, response_body, completed_at)`. On a repeat request with the same key:

- **In-flight:** the second request blocks on the first's completion (a sync.Event keyed on the idempotency key) and returns its response.
- **Within TTL:** returns the cached response. Does NOT re-run the model. Does NOT count against any quota.
- **Expired or evicted:** re-runs.

`call_index` increments only when the orchestrator deliberately wants a different result (e.g., `regenerate_with_new_seed` action). It is **not** a retry counter — retries reuse the same key.

The orchestrator generates keys deterministically from persisted state, so a backend that crashed-and-restarted produces the same key. This is what makes "never pay twice" hold across crashes.

## 3. Orchestrator ↔ Blender (subprocess)

Not HTTP. Subprocess with JSON-on-stdout protocol.

```bash
{BLENDER_EXE} --background --factory-startup \
              --python blender/scripts/{script_name}.py -- \
              --input ... --output ... --json-progress
```

Each Blender script:

- Reads its CLI args after `--`.
- Emits one JSON object per line on **stderr** (stdout is reserved for Blender's own noise): `{"event": "progress", "progress": 0.4, "msg": "..."}`.
- Final line on stderr: `{"event": "done", "ok": true, "outputs": [...], "stats": {...}}` or `{"event": "done", "ok": false, "error": "..."}`.
- Exit code: 0 on success, non-zero on failure.

The orchestrator's `blender_runner.py` tails stderr line-by-line and converts events into SSE `stage_update`s.

## 4. File-system contracts

- `temp/{job_id}/input/` — preprocessed inputs.
- `temp/{job_id}/iter_{n}/{stage}/` — per-stage artifacts.
- `exports/{job_id}/` — final deliverables. Never deleted automatically.
- `logs/` — rotating logs.
- `models_cache/` — HuggingFace cache (env `HF_HOME=./models_cache`).

All paths in HTTP responses are relative to repo root.

---

## 5. Versioning

The HTTP API is versionless in v1 (single-user, single-binary). When we ship updates that break the contract, the Tauri shell will refuse to start an incompatible backend. Mechanism: backend `/health` returns `{api_version: "1.0"}`; Tauri pins the major version it expects in `tauri.conf.json`.

---

## 6. What is NOT in the contract (intentional)

- No mutation of the input image. Tauri uploads paths; Python reads them in place.
- No streaming of mesh bytes. Meshes go through the file system, not the HTTP body, even though the API allows base64. (Base64 is only used for the orchestrator ↔ remote GPU hop, where there is no shared filesystem.)
- No "subscribe to all jobs" event stream. One SSE per job. Keeps state simple.
